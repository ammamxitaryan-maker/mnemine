## 🚀 Как использовать

### Локальная разработка
```bash
# 1. Установка зависимостей
pnpm install

# 2. Настройка окружения (опционально)
cp env.example .env.local

# 3. Запуск
pnpm dev
```

**Результат:**
- Frontend: http://localhost:5173
- Backend: http://localhost:10112
- Автоматический вход как тестовый пользователь
- Кнопка "🔧 Dev Auth" для переключения пользователей

### Деплой на Render
```bash
# 1. Настройка переменных окружения в Render Dashboard
# 2. Деплой
render deploy

# Или автоматический деплой с проверками
node scripts/deploy-to-render.js
```

## 🔧 Особенности реализации

### Fallback авторизация
- **Автоматическое определение** - проверяет `window.location.hostname`
- **Безопасность** - работает только в локальной разработке
- **Гибкость** - поддерживает переключение между пользователями
- **Совместимость** - не влияет на продакшн код

### Dev сервер
- **Concurrently** - запускает клиент и сервер одновременно
- **Проксирование** - Vite автоматически проксирует API запросы
- **HMR** - React Fast Refresh для мгновенных обновлений
- **TypeScript** - автоматическая компиляция и перезапуск сервера

### Продакшн сборка
- **Оптимизация** - минификация и tree-shaking
- **Статические файлы** - фронтенд копируется в сервер/public
- **Переменные окружения** - отдельные настройки для продакшна
- **Безопасность** - fallback авторизация отключена
