# 🚀 Quick Deploy Scripts

Быстрые скрипты для автоматического коммита и пуша изменений.

## 📁 Доступные скрипты:

### 1. **quick-deploy.bat** (Windows - простой)
```bash
quick-deploy.bat
```
- Добавляет все файлы: `git add .`
- Коммитит с сообщением "1234567": `git commit -m "1234567"`
- Пушит в main: `git push origin main`

### 2. **git-acp.bat** (Windows - с временной меткой)
```bash
git-acp.bat
```
- Добавляет все файлы
- Коммитит с временной меткой: `git commit -m "Auto-commit 2025-10-02 16:30:45"`
- Пушит в main
- Показывает детальную информацию

### 3. **git-acp.ps1** (PowerShell)
```bash
.\git-acp.ps1
```
- То же что git-acp.bat, но для PowerShell
- Цветной вывод
- Лучшая обработка ошибок

### 4. **git-acp.sh** (Linux/Mac)
```bash
./git-acp.sh
```
- Для Unix-систем
- С временной меткой
- Детальная информация

## 🎯 Как использовать:

### Windows:
1. **Простой способ**: Двойной клик на `quick-deploy.bat`
2. **С временной меткой**: Двойной клик на `git-acp.bat`
3. **PowerShell**: Правый клик → "Запустить с PowerShell" на `git-acp.ps1`

### Linux/Mac:
```bash
./git-acp.sh
```

## ⚡ Быстрый деплой:

1. Внесите изменения в код
2. Запустите любой из скриптов
3. Готово! Изменения автоматически задеплоены

## 🔧 Настройка:

Если хотите изменить сообщение коммита в `quick-deploy.bat`:
```batch
git commit -m "Ваше сообщение"
```

## 📋 Что делают скрипты:

1. ✅ `git add .` - добавляют все изменения
2. ✅ `git commit -m "..."` - создают коммит
3. ✅ `git push origin main` - пушат в репозиторий
4. ✅ Показывают результат каждой операции

## 🚨 Важно:

- Убедитесь, что вы в правильной директории проекта
- Проверьте, что у вас есть права на push в репозиторий
- Скрипты автоматически добавят ВСЕ изменения (включая неотслеживаемые файлы)
