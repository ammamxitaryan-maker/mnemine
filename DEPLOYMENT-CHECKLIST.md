# 🚀 Mnemine Deployment Checklist

## Pre-Deployment Checklist

### ✅ Code Quality
- [ ] All TypeScript types are valid (`pnpm run type-check`)
- [ ] Build completes successfully (`pnpm run build`)
- [ ] Deployment verification passes (`pnpm run verify:deployment`)
- [ ] No linting errors (`pnpm run lint`)
- [ ] Tests pass (`pnpm run test`)

### ✅ Environment Variables
- [ ] `TELEGRAM_BOT_TOKEN` - Your Telegram bot token
- [ ] `ADMIN_TELEGRAM_ID` - Your Telegram user ID (e.g., 6760298907)
- [ ] `JWT_SECRET` - 32+ character secret (auto-generated by Render)
- [ ] `ENCRYPTION_KEY` - 32 character key (auto-generated by Render)
- [ ] `SESSION_SECRET` - Session secret (auto-generated by Render)

### ✅ Database
- [ ] PostgreSQL database created in Render
- [ ] Database URL properly configured
- [ ] Prisma schema is up to date
- [ ] Database migrations ready

### ✅ Configuration Files
- [ ] `render.yaml` is properly configured
- [ ] Environment variables are set in Render dashboard
- [ ] Build and start commands are correct
- [ ] Health check endpoint configured

## Deployment Steps

### 1. Repository Setup
- [ ] Code is pushed to GitHub main branch
- [ ] Repository is connected to Render
- [ ] Blueprint deployment is configured

### 2. Render Service Configuration
- [ ] Web service created with correct settings
- [ ] PostgreSQL database service created
- [ ] Environment variables configured
- [ ] Build command: `pnpm install --frozen-lockfile && pnpm run build && pnpm run copy:frontend`
- [ ] Start command: `pnpm run start`
- [ ] Health check path: `/health`

### 3. Environment Variables Setup
```
NODE_ENV=production
PORT=10000
DATABASE_URL=<from database service>
JWT_SECRET=<auto-generated>
TELEGRAM_BOT_TOKEN=<your token>
ADMIN_TELEGRAM_ID=<your telegram id>
ENCRYPTION_KEY=<auto-generated>
SESSION_SECRET=<auto-generated>
CORS_ORIGIN=https://your-app-name.onrender.com
FRONTEND_URL=https://your-app-name.onrender.com
ALLOWED_ORIGINS=https://your-app-name.onrender.com
VITE_BACKEND_URL=https://your-app-name.onrender.com
VITE_WS_URL=wss://your-app-name.onrender.com/ws
VITE_APP_NAME=Mnemine
RATE_LIMIT_WINDOW_MS=900000
RATE_LIMIT_MAX_REQUESTS=100
LOG_LEVEL=info
```

### 4. Deployment Execution
- [ ] Trigger deployment in Render dashboard
- [ ] Monitor build logs for errors
- [ ] Verify deployment completes successfully
- [ ] Check service health status
