## 🚀 **Инструкции для деплоя**

1. **Создать сервис в Render:**
   - Выбрать "Web Service"
   - Подключить GitHub репозиторий
   - Выбрать корневую папку проекта

2. **Настроить переменные окружения:**
   - `TELEGRAM_BOT_TOKEN` - токен бота (получить у @BotFather)
   - Остальные переменные уже настроены в `render.yaml`

3. **Деплой:**
   - Render автоматически выполнит команды из `render.yaml`
   - Сборка: `pnpm install --frozen-lockfile && pnpm run build:prod && pnpm run copy:frontend`
   - Запуск: `node dist/server/src/index.js`

## ⚠️ **Важные моменты**

- **TELEGRAM_BOT_TOKEN** нужно настроить вручную в Render Dashboard
- **База данных** будет создана автоматически (PostgreSQL)
- **Webhook URL** будет: `https://your-app-name.onrender.com/api/webhook/YOUR_BOT_TOKEN`
- **Фронтенд** будет доступен по основному URL сервиса

## 🔍 **Проверка после деплоя**

1. **Основная страница:**
   - Открыть URL сервиса - должен загрузиться фронтенд
   - Проверить `/health` endpoint - должен вернуть статус

2. **Динамическая загрузка:**
   - Перейти на `/slots` - должна загрузиться страница слотов
   - Перейти на `/referrals` - должна загрузиться страница рефералов
   - Перейти на `/admin` - должна загрузиться админ панель
   - Обновить страницу - должна остаться на том же роуте

3. **API и статика:**
   - Проверить `/api/health` - должен вернуть API ответ
   - Проверить `/assets/` - должны загружаться CSS/JS файлы
   - Проверить `/locales/` - должны загружаться файлы переводов

4. **Логи и мониторинг:**
   - Проверить логи в Render Dashboard
   - Протестировать Telegram WebApp интеграцию

## 📞 **Поддержка**

Если возникнут проблемы:
1. Проверить логи в Render Dashboard
2. Убедиться, что все переменные окружения настроены
3. Проверить, что `TELEGRAM_BOT_TOKEN` правильный
4. Проверить подключение к базе данных
